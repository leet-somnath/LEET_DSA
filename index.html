AI-900 Exam Preparation Guide

1. Introduction to AI

1.1 What is Artificial Intelligence?

  • Definition of AI: Simulation of human intelligence in machines designed to think and act like humans.
  • Types of AI:
    • Narrow AI: Specialized in a single task (e.g., speech recognition).
    • General AI: Can perform any intellectual task that a human can.
    • Super AI: Surpasses human intelligence in all aspects.
  • Applications of AI:
    • Healthcare: Diagnostic systems, personalized medicine.
    • Finance: Fraud detection, algorithmic trading.
    • Retail: Personalized recommendations, inventory management.
    • Transportation: Autonomous vehicles, traffic management.

1.2 Principles of Machine Learning

  • Definition and Types of Machine Learning:
    • Supervised Learning: Uses labeled data to train models (e.g., regression, classification).
    • Unsupervised Learning: Identifies patterns in unlabeled data (e.g., clustering, association).
    • Reinforcement Learning: Learns by interacting with the environment to maximize rewards.
  • Machine Learning Process:
    • Data Collection: Gathering data relevant to the problem.
    • Data Preparation: Cleaning and preprocessing data.
    • Model Training: Using algorithms to train the model on the data.
    • Model Evaluation: Assessing the model’s performance using metrics.
    • Model Deployment: Integrating the model into a production environment.

2. Azure AI Services

2.1 Overview of Azure AI Services

  • Introduction to Azure AI Platform: A suite of AI services and tools for building and deploying AI solutions.
  • Categories of Azure AI Services:
    • Vision: Image and video processing (e.g., Computer Vision, Face API).
    • Speech: Speech recognition and synthesis (e.g., Speech-to-Text, Text-to-Speech).
    • Language: Natural language understanding (e.g., Text Analytics, Translator).
    • Decision: Making informed decisions (e.g., Personalizer, Anomaly Detector).
    • Search: Enhanced search capabilities (e.g., Azure Search).

2.2 Azure Machine Learning

  • Key Features of Azure Machine Learning: End-to-end machine learning lifecycle management, automated ML, drag-and-drop designer, Python SDK.
  • Creating and Managing Machine Learning Models:
    • Building models using Azure Machine Learning Studio.
    • Utilizing Jupyter notebooks for advanced model creation.
  • Automated ML and Its Advantages: Automates the process of model selection, hyperparameter tuning, and deployment, saving time and improving efficiency.

2.3 Cognitive Services

  • Overview of Cognitive Services: Pre-built AI capabilities that can be easily integrated into applications.
  • Categories and Practical Use Cases:
    • Vision: Object detection in images, OCR (Optical Character Recognition).
    • Speech: Transcribing speech to text, generating natural-sounding speech.
    • Language: Sentiment analysis, language translation.
    • Decision: Personalized recommendations, detecting anomalies in data.

2.4 Azure Bot Service

  • Introduction to Azure Bot Service: Platform for building and deploying conversational agents.
  • Creating a Simple Bot:
    • Using the Bot Framework SDK.
    • Integrating with various channels (e.g., Microsoft Teams, Slack).
  • Integration with Other AI Services: Combining bots with Cognitive Services for enhanced capabilities (e.g., language understanding with LUIS).

3. Machine Learning on Azure

3.1 Machine Learning Models

  • Types of Machine Learning Models:
    • Classification: Categorizing data into predefined classes (e.g., spam detection).
    • Regression: Predicting continuous values (e.g., house prices).
    • Clustering: Grouping similar data points together (e.g., customer segmentation).

3.2 Training and Evaluating Models

  • Training Datasets and Testing Datasets: Dividing data into training and testing sets to validate model performance.
  • Model Evaluation Metrics:
    • Accuracy: Proportion of correctly predicted instances.
    • Precision: Proportion of true positive predictions out of all positive predictions.
    • Recall: Proportion of true positive predictions out of all actual positives.
    • F1 Score: Harmonic mean of precision and recall, providing a balanced measure.
  • Cross-Validation Techniques: Splitting data into multiple folds to ensure robustness in evaluation.

3.3 Deploying Machine Learning Models

  • Deploying Models Using Azure Machine Learning:
    • Creating and configuring inference pipelines.
    • Deploying models as web services.
  • Managing and Monitoring Deployed Models: Tracking model performance, managing versioning, and handling scaling issues.

4. Computer Vision

4.1 Computer Vision Concepts

  • Introduction to Computer Vision: Enabling machines to interpret and make decisions based on visual data.
  • Key Tasks:
    • Image Classification: Assigning labels to images based on content.
    • Object Detection: Identifying and locating objects within images.
    • Image Segmentation: Dividing an image into segments for detailed analysis.

4.2 Azure Computer Vision Service

  • Features of Azure Computer Vision: Image analysis, face detection, OCR.
  • Using the Computer Vision API:
    • Analyzing images to extract information (e.g., tags, descriptions).
    • Detecting objects and faces in images.
  • Custom Vision Service: Building custom image classification models tailored to specific needs.

5. Natural Language Processing (NLP)

5.1 NLP Concepts

  • Introduction to NLP: Enabling machines to understand, interpret, and generate human language.
  • Key NLP Tasks:
    • Sentiment Analysis: Determining the sentiment expressed in text.
    • Language Translation: Translating text from one language to another.
    • Text Summarization: Condensing long texts into concise summaries.

5.2 Azure Language Services

  • Features of Azure Text Analytics: Sentiment analysis, key phrase extraction, language detection.
  • Using the Text Analytics API:
    • Analyzing text for sentiment.
    • Extracting key phrases and identifying languages.
  • Azure Translator: Translating text across multiple languages with high accuracy.

6. Conversational AI

6.1 Conversational AI Concepts

  • Introduction to Conversational AI: Building systems that can engage in human-like dialogue.
  • Key Components:
    • Intent: Purpose behind a user’s input.
    • Entity: Specific pieces of information relevant to the intent.
    • Dialogue: Flow of conversation to handle user interactions.

6.2 Azure Bot Service

  • Creating and Deploying Bots:
    • Using the Bot Framework Composer for no-code/low-code bot development.
    • Deploying bots to various channels and managing them.
  • Integrating with QnA Maker:
    • Creating a knowledge base from FAQs and documentation.
    • Building and deploying bots that provide answers from the knowledge base.

7. Responsible AI

7.1 Principles of Responsible AI

  • Fairness: Ensuring AI systems do not favor any group.
  • Reliability and Safety: Building systems that operate safely and reliably.
  • Privacy and Security: Protecting user data and maintaining privacy.
  • Inclusiveness: Designing AI systems that are accessible to all.
  • Transparency: Making AI operations understandable.
  • Accountability: Holding systems and their creators accountable for outcomes.

7.2 Implementing Responsible AI on Azure

  • Tools and Services:
    • Fairlearn: Assessing and mitigating unfairness in machine learning models.
    • InterpretML: Interpreting model predictions to understand decision-making processes.
  • Best Practices:
    • Incorporating diverse datasets to avoid bias.
    • Regularly monitoring AI systems for performance and fairness.
    • Engaging stakeholders in the design and deployment of AI systems.
